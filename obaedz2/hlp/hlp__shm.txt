~005 Первый ~000прямоугольник определения загрузки компьютера имеет следующие цветовые границы:
~000 желтый  - легкая  работа   (программа работает не мешая работе других программ)
~000 зеленый - слабая  загрузка (программа работает нормально без загрузки других программ)
~000 синий   - средняя загрузка (программа может немного притормаживать,но работать можно)
~000 красный - сильная загрузка (компьютер не подходит для работы с программой)
~000                             
~005 Второй ~000прямоугольник определяет включен или нет драйвер обмена данными по COM порту.
~000 Голубой - включен, красный - выключен.
~000 Драйвер может быть выключен, если: 
~000 - в конфигурации определен для обмена COM2, а на компьютере установлен только COM1;
~000 - параллельно работает программа, которая использует этот же COM порт;
~000 - COM порт имеет нестандартные настройки;
~000 
~005 Третий ~000прямоугольник определяет включен или нет драйвер обмена данными по NetBIOS.
~000 Голубой - включен, красный - выключен.
~000 Драйвер может быть выключен, если: 
~000 - на компьютере не установлен EtherNet адаптер через который ожидается работа NetBIOS;
~000 - неправильно указаны настройки адаптера Ethernet (IRQ, BaseIO...);
~000 - в Windows-е обмен данными через драйвер NetBIOS не поддерживается, хотя прямоугольник
~000   может выводится голубым цветом.
~000 
~005 Линия на клетчатом поле ~000показывает какой драйвер обмена работает в данный момент.
~000 Зеленая линия на синем клетчатом поле  - обмен данными с контроллером по COM порту.
~000 Синяя линия на зеленом клетчатом поле  - обмен данными с контроллером по NetBIOS.
~000 Нет линии,сине-зеленое клетчатое поле  - нет обмена данными или сильно медленный.
~000 Синяя и зеленая линии  на белом фоне   - обмен данными по COM порту и по NetBIOS.
~000 
~000 При запуске программы, для обмена данными между  контроллером и рабочей станцией
~000 оператора, включается сеть NetBIOS. 
~000 Если в процессе работы программы, в течении 2х секунд не будет обмена данными по 
~000 сети NetBIOS, обмен переключается на COM порт.  
~000 Если после этого, сеть NetBIOS опять появится - обмен данными переключится на 
~000 драйвер NetBIOS и т.д..
~000 Переодичность и равномерность обмена данными можно посмотреть на экране "~004СИСТЕМА~000"
